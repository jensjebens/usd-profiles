UsdProfiles implements the USD-Profiles proposal, providing a declarative capability taxonomy for OpenUSD. It enables applications and assets to declare, discover, and validate functional capabilities through profiles and a capability directed acyclic graph (DAG).

Purpose and Motivation

USD's flexibility via custom schemas, proprietary formats, and extensions makes it applicable across many industries, but that flexibility introduces interoperability challenges. Applications implement different subsets of USD schemas, leading to inconsistencies when content moves between tools.

UsdProfiles addresses this by enabling applications to:

  • Discover and validate a document's compatibility and runtime requirements
  • Assess interoperability with particular scenes or scene elements
  • Determine capability satisfaction for declared functional needs

Rather than relying on ad hoc extension discovery, UsdProfiles introduces a standardized, declarative approach.

Core Concepts

Capabilities

A capability is an atomic unit of functionality, identified by a token using reverse domain notation:

  • usd — base USD capability (root of the DAG)
  • usd.core — core USD interchange capabilities
  • com.nvidia.simready.geom — NVIDIA SimReady geometry capabilities
  • com.nvidia.simready.physics.rigidBodies — rigid body physics

Capabilities are organized in a directed acyclic graph (DAG) where edges represent "depends on" relationships (called predecessors). Every capability transitively depends on the usd root.

usd (root)
├── usd.core
│ └── usd.core.v25_05 [PROFILE]
├── aousd.interchange
│ └── aousd.interchange.v1_0 [PROFILE]
└── com.nvidia.simready
├── com.nvidia.simready.geom
├── com.nvidia.simready.physics
│ └── com.nvidia.simready.physics.rigidBodies
└── com.nvidia.simready.prop_robotics_neutral [PROFILE]

Profiles

A profile is a capability node tagged with "isProfile": true in the capability DAG. It represents a coherent, named set of functionality suitable for a specific use case. Examples:

  • usd.core.v25_05 — OpenUSD 25.05 baseline
  • aousd.interchange.v1_0 — AOUSD interchange requirements
  • com.nvidia.simready.prop_robotics_neutral — neutral robotics prop for simulation

A profile's full capability set is determined by its transitive predecessors in the DAG. For example, com.nvidia.simready.prop_robotics_neutral expands to:

  • com.nvidia.simready.geom
  • com.nvidia.simready.physics.rigidBodies
  • com.nvidia.simready.physics
  • com.nvidia.simready
  • usd

Naming Convention

Capabilities use reverse domain notation for hierarchical clarity:

  • Canonical capabilities: usd.* — maintained by OpenUSD/AOUSD
  • Vendor extensions: com.nvidia.simready.* — NVIDIA SimReady
  • Custom extensions: com.example.myapp.* — your organization

All vendor capabilities must transitively inherit from usd. This ensures that any asset's capability requirements can always be traced back to the root, enabling clear compatibility reporting.

ProfileAPI Schema

Overview

The ProfileAPI is a single-apply API schema with two attributes:

  • profiles:profile (token) — the profile identifier this prim conforms to
  • profiles:capabilities (token[]) — additional explicit capability declarations

Authoring

from pxr import Usd, UsdGeom, UsdProfiles
stage = Usd.Stage.CreateNew("my_asset.usda")
root = UsdGeom.Xform.Define(stage, "/MyRobot")
# Apply ProfileAPI and set a profile
api = UsdProfiles.ProfileAPI.Apply(root.GetPrim())
api.GetProfileAttr().Set("com.nvidia.simready.prop_robotics_neutral")
# Optionally declare additional capabilities
api.CreateCapabilitiesAttr([
"com.nvidia.simready.physics.rigidBodies",
"com.nvidia.simready.geom",
])
stage.Save()

The resulting USD file:

#usda 1.0
def Xform "MyRobot" (
prepend apiSchemas = ["ProfileAPI"]
)
{
token[] profiles:capabilities = [
"com.nvidia.simready.physics.rigidBodies",
"com.nvidia.simready.geom"
]
token profiles:profile = "com.nvidia.simready.prop_robotics_neutral"
}

Resolution Semantics

ProfileAPI follows a "nearest \b ancestor \b wins" resolution rule:

  • If a prim has ProfileAPI applied with a non-empty profiles:profile, that profile governs the prim and its descendants.
  • If a prim does not have ProfileAPI, the nearest ancestor's profile applies.
  • Capability requirements do not propagate upward through hierarchies.

This enables strategic authoring at traversal boundaries for performance:

/Warehouse ← ProfileAPI: "com.nvidia.simready.prop_robotics_neutral"
/Warehouse/ShelfUnit_A ← (inherits from /Warehouse)
/Warehouse/ShelfUnit_A/Frame ← (inherits from /Warehouse)
/Warehouse/RobotArm ← ProfileAPI: "com.nvidia.simready.prop_robotics_neutral"
/Warehouse/RobotArm/Gripper ← (inherits from /RobotArm)
# Query effective profile — walks ancestor chain
profile = UsdProfiles.GetEffectiveProfile(
stage.GetPrimAtPath("/Warehouse/ShelfUnit_A/Frame"))
# → "com.nvidia.simready.prop_robotics_neutral" (inherited from /Warehouse)

Capability Registry

Overview

The CapabilityRegistry is a singleton that collects capability declarations from all registered plugins at startup. Capabilities are declared in plugInfo.json:

{
"Plugins": [{
"Info": {
"Capabilities": {
"com.example.myCapability": {
"docstring": "Description of my capability",
"predecessors": ["usd"],
"isProfile": false
},
"com.example.myProfile": {
"docstring": "A profile combining capabilities",
"predecessors": [
"com.example.myCapability",
"com.nvidia.simready.geom"
],
"isProfile": true
}
}
}
}]
}

Querying the Registry

registry = UsdProfiles.CapabilityRegistry.GetInstance()
# Check membership
registry.IsCapability("com.nvidia.simready.geom") # True
registry.IsProfile("com.nvidia.simready.prop_robotics_neutral") # True
# Navigate the DAG
registry.GetPredecessors("com.nvidia.simready.geom")
# → ["com.nvidia.simready"]
registry.GetTransitivePredecessors(
"com.nvidia.simready.prop_robotics_neutral")
# → ["com.nvidia.simready.geom",
# "com.nvidia.simready.physics.rigidBodies",
# "com.nvidia.simready.physics",
# "com.nvidia.simready",
# "usd"]
# List all
registry.GetAllCapabilities() # all registered capability tokens
registry.GetAllProfiles() # only those tagged as profiles

Extending the Registry

Any USD plugin can declare additional capabilities in its plugInfo.json. The registry loads from all plugins, so capabilities from multiple sources are merged into a single DAG. This enables:

  • OpenUSD core to declare canonical usd.* capabilities
  • AOUSD to declare interchange aousd.* capabilities
  • Vendors to declare domain-specific capabilities (com.nvidia.simready.*)
  • Studios to declare project-specific capabilities (com.studio.project.*)

Profile Registry

The ProfileRegistry is a convenience layer that resolves profiles to their full capability sets:

preg = UsdProfiles.ProfileRegistry.GetInstance()
preg.GetAllProfiles()
preg.GetProfileCapabilities("com.nvidia.simready.prop_robotics_neutral")
# → ["com.nvidia.simready.prop_robotics_neutral",
# "com.nvidia.simready.geom",
# "com.nvidia.simready.physics.rigidBodies",
# "com.nvidia.simready.physics",
# "com.nvidia.simready",
# "usd"]

Convenience Query Functions

Two free functions provide the most common query patterns:

  • UsdProfilesGetEffectiveProfile() — Walk the ancestor chain and return the nearest authored profile token.
  • UsdProfilesGetEffectiveCapabilities() — Return the full capability set implied by the effective profile, plus any explicit profiles:capabilities on the authoring prim.
# Get the profile governing this prim
profile = UsdProfiles.GetEffectiveProfile(prim)
# Get all capabilities this prim satisfies
caps = UsdProfiles.GetEffectiveCapabilities(prim)

Full Example: Robotics Warehouse

from pxr import Usd, UsdGeom, UsdProfiles
# Create a warehouse scene
stage = Usd.Stage.CreateNew("warehouse.usda")
stage.SetMetadata("upAxis", "Z")
stage.SetMetadata("metersPerUnit", 1.0)
# Warehouse root — declares SimReady profile
warehouse = UsdGeom.Xform.Define(stage, "/Warehouse")
api = UsdProfiles.ProfileAPI.Apply(warehouse.GetPrim())
api.GetProfileAttr().Set("com.nvidia.simready.prop_robotics_neutral")
# Shelving (inherits profile from warehouse)
shelf = UsdGeom.Xform.Define(stage, "/Warehouse/ShelfUnit_A")
frame = UsdGeom.Xform.Define(stage, "/Warehouse/ShelfUnit_A/Frame")
# Robot arm (own profile with explicit capabilities)
robot = UsdGeom.Xform.Define(stage, "/Warehouse/RobotArm")
robot_api = UsdProfiles.ProfileAPI.Apply(robot.GetPrim())
robot_api.GetProfileAttr().Set(
"com.nvidia.simready.prop_robotics_neutral")
robot_api.CreateCapabilitiesAttr([
"com.nvidia.simready.physics.rigidBodies",
"com.nvidia.simready.geom",
])
gripper = UsdGeom.Xform.Define(stage, "/Warehouse/RobotArm/Gripper")
stage.Save()
# --- Queries ---
# Effective profiles (nearest ancestor wins)
for path in ["/Warehouse", "/Warehouse/ShelfUnit_A/Frame",
"/Warehouse/RobotArm/Gripper"]:
prim = stage.GetPrimAtPath(path)
profile = UsdProfiles.GetEffectiveProfile(prim)
print(f"{path}: {profile}")
# Full capability set
caps = UsdProfiles.GetEffectiveCapabilities(
stage.GetPrimAtPath("/Warehouse/RobotArm"))
print(f"Robot capabilities: {[str(c) for c in caps]}")
# Explore the registry
reg = UsdProfiles.CapabilityRegistry.GetInstance()
for cap in reg.GetAllProfiles():
preds = reg.GetTransitivePredecessors(str(cap))
print(f"Profile {cap} implies: {[str(p) for p in preds]}")

Future Directions

The current implementation covers explicit capability queries via ProfileAPI. Planned extensions include:

  • Introspective queries — infer capabilities from applied schemas (e.g. UsdPhysicsRigidBodyAPI implies com.nvidia.simready.physics.rigidBodies)
  • Traversal pruning — use ProfileAPI boundaries to optimize scene traversal
  • UsdValidation integration — select validators based on profile requirements
  • Build integration — guide OpenUSD builds based on capability profiles
